## 2.2 舆情分析平台

- 项目目标：实现一个面向指定主题的舆情分析平台，从公开信息、多媒体内容、内部沉淀数据三个方向并行收集信息，汇总生成结构化舆情报告
- 交付形式：提供可运行 Web 控制台、分析服务、配置文件、示例数据、接口说明和至少一个完整主题的分析案例

### 2.2.1 开发范围

- 控制台能力：系统启停、单功能应用启停、状态查看、输出查看、配置管理
- 分析能力：主题检索、多媒体检索、论坛采集、洞察分析、报告生成、图谱查询
- 展示能力：应用输出查看、论坛日志查看、图谱查看器、系统状态面板

### 2.2.2 用户角色

- `操作用户`：启动和停止单功能应用、发起主题检索、查看输出结果
- `报告查看人`：查看已生成报告、图谱结果、论坛日志和来源证据
- `系统管理员`：维护系统配置、查看系统状态、启停整体服务、查看运行日志

### 2.2.3 页面需求

- `控制台首页`：展示各单功能应用状态、启动按钮、停止按钮、最近输出摘要和系统状态
- `主题检索页`：输入主题词、关键词或检索条件，发起主题分析并查看文本输出
- `论坛监控页`：展示论坛采集启动状态、最新日志、历史日志查询入口
- `图谱查看页`：展示最新图谱和指定报告图谱，支持节点查询和关系查看
- `配置管理页`：展示系统配置、采集配置和运行参数，支持读取与保存
- `系统状态页`：展示整体服务状态、各应用状态、最近错误信息和资源运行概况

### 2.2.4 任务状态与执行流程

- 单功能应用状态至少包含 `stopped`、`starting`、`running`、`stopping`、`failed`
- 论坛采集状态至少包含 `idle`、`running`、`stopped`、`failed`
- 系统状态至少包含 `offline`、`starting`、`online`、`shutting_down`
- 系统启动后，允许独立启动主题检索、多媒体检索、洞察分析等单功能应用
- 每个单功能应用都要单独保存运行状态、标准输出和错误输出
- 停止应用后，不再继续接收新任务，但已生成的输出结果和日志保留
- 图谱数据和报告结果要支持按报告编号回查

### 2.2.5 前端模块要求

- `控制台模块`：处理应用状态查询、应用启停、系统状态展示
- `输出模块`：处理单功能应用输出查看、测试日志查看、滚动展示
- `论坛模块`：处理论坛任务启停、论坛日志查看、历史日志查询
- `图谱模块`：处理图谱查看、报告切换、图查询结果展示
- `检索模块`：处理主题检索请求、结果展示和来源查看
- `配置模块`：处理配置读取、更新、回显

### 2.2.6 后端模块要求

- `控制层模块`：负责 Web 路由、页面渲染、状态接口和系统启停入口
- `进程管理模块`：负责单功能应用启动、停止、状态采集、输出采集
- `检索分析模块`：负责主题检索、内容分析、洞察生成和报告汇总
- `论坛模块`：负责论坛采集任务控制、日志输出和历史记录保存
- `图谱模块`：负责图谱结果加载、指定报告查询和关系检索
- `配置模块`：负责配置读取、保存和运行参数热更新

### 2.2.7 数据存储要求

- `config.json`：保存系统配置、运行参数和默认应用开关
- `runtime/apps/{app_name}.log`：保存各单功能应用标准输出和错误输出
- `runtime/forum/latest.log`：保存论坛采集最新运行日志
- `runtime/forum/history/{date}.json`：保存论坛日志历史记录
- `reports/{report_id}/report.md`：保存结构化分析报告
- `graphs/{report_id}/graph.json`：保存图谱节点和边关系数据
- `outputs/{app_name}/latest.txt`：保存单功能应用最近一次文本输出结果

### 2.2.8 采集与分析规则

- 每次主题检索都必须至少保存主题词、执行时间、来源摘要和分析输出
- 支持来源类型至少包含 `news`、`image`、`video`、`forum_post`、`internal_data`
- 图谱结果至少保存节点标识、节点类型、关系类型和来源引用
- 论坛日志至少保留时间、事件类型、消息内容和任务状态
- 最终报告中的结论必须可以追溯到来源内容、论坛记录或图谱节点

### 2.2.9 接口要求

- `GET /api/status`：查询控制台状态和各应用运行状态
- `GET /api/start/{app_name}`：启动指定单功能应用
- `GET /api/stop/{app_name}`：停止指定单功能应用
- `GET /api/output/{app_name}`：查询指定应用最近输出
- `GET /api/test_log/{app_name}`：查询指定应用测试日志
- `POST /api/search`：请求字段 `query`、`source_types`，返回主题检索和分析结果
- `GET /api/forum/start`：启动论坛采集
- `GET /api/forum/stop`：停止论坛采集
- `GET /api/forum/log`：查询论坛最新日志
- `POST /api/forum/log/history`：按日期查询论坛历史日志
- `GET /api/config`：查询系统配置
- `POST /api/config`：更新系统配置
- `GET /api/system/status`：查询整体服务状态
- `POST /api/system/start`：启动整体服务
- `POST /api/system/shutdown`：关闭整体服务
- `GET /api/graph/latest`：查询最新图谱结果
- `GET /api/graph/{report_id}`：查询指定报告图谱
- `POST /api/graph/query`：按节点或关系条件查询图谱
- `GET /graph-viewer`：进入图谱查看页
- `GET /graph-viewer/{report_id}`：进入指定报告图谱查看页

### 2.2.10 实时消息类型

- `app_status`：字段至少包含 `app_name`、`status`
- `app_output`：字段至少包含 `app_name`、`output_text`
- `forum_log`：字段至少包含 `message_text`、`task_status`
- `system_status`：字段至少包含 `system_status`、`running_apps`
- `graph_ready`：字段至少包含 `report_id`、`graph_path`
- `error`：字段至少包含 `module_name`、`error_message`

### 2.2.11 报告输出格式

- `事件概述`：说明主题背景和分析范围
- `时间线`：按时间顺序列出关键事件
- `传播渠道`：按来源类型统计传播分布
- `主要观点`：列出不同来源和不同角色总结出的核心观点
- `情绪倾向`：给出正向、中性、负向的整体判断
- `风险判断`：列出高风险点和关注事项
- `重点证据`：列出来源标题、链接、摘要和引用位置
- `结论摘要`：给出最终结论和建议

### 2.2.12 验收标准

- 能通过控制台启动和停止单功能应用
- 能查看应用状态、应用输出和最近错误信息
- 能启动论坛采集并查看最新日志和历史日志
- 能发起主题检索并看到结构化分析结果
- 能查看最新图谱和指定报告图谱
- 能更新配置并即时生效
- 能完成至少一个完整主题的检索、分析、图谱查看和结果展示演示
